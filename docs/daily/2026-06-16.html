🛠️ Vercel Generative UI × Agent Harness
一句话

一个把 Coding Agent 执行过程渲染成结构化 UI 的示例 Harness，让 Agent 不只输出 Markdown，而是输出步骤、Diff、终端、测试结果和图表组件。

链接

https://github.com/vercel-labs/json-render/tree/main/examples/harness-chat ↗

怎么玩

打开 GitHub 仓库后进入 examples/harness-chat 目录，先阅读 README 和 package 脚本，按项目要求安装依赖并启动本地开发服务。核心玩法是把 Claude Code、Codex、Pi 这类 Coding Agent 放进 Vercel Sandbox，让它真实改代码、跑命令、测用例；汇报阶段则让 Agent 按 json-render 的受约束 JSON UI 规格返回内容。你可以先跑官方示例，再把自己的 Agent 输出从纯文本改成 JSON 组件，比如 terminal、diff、test-result、chart 等模块，观察前端如何实时渲染。

为什么值得关注

Agent 产品的瓶颈不只是「会不会做事」，还包括用户如何理解、审查和信任它做了什么。这个示例把执行轨迹 UI 化，等于给 Coding Agent 加了一层可观察、可交互的报告协议，适合用来做内部开发助手、代码审查面板或任务执行控制台。

应用场景
  • 给自研 Coding Agent 做实时执行看板，展示命令、补丁和测试结果。
  • 把 Agent 工作流产品从聊天窗口升级为可审查的工程控制台。

🛠️ Kami
一句话

Tw93 开源的 AI 写作与排版工具，用来生成更整洁的简历、报告和多语言文档。

链接

https://github.com/tw93/Kami ↗

怎么玩

进入 GitHub 仓库后先查看 README 的安装方式和示例截图，按项目说明准备运行环境。使用时可以把已有简历、项目介绍或报告草稿交给 Kami，先让它做事实核查和结构重写，再选择目标语言和页面样式生成两页以内的整洁内容。建议从一个真实但不重要的简历副本开始测试：先输入原始经历，再对比重写前后的事实、措辞、长度和版式，确认没有虚构经历后再导出或复制到正式文档。

为什么值得关注

很多 AI 写作工具只负责「生成文字」，Kami 更强调最终文档的可读性、事实检查和极简设计系统。它适合把 LLM 写作结果落到真实交付物上，尤其是简历、汇报、申请材料这类既要准确信息又要视觉干净的内容。

应用场景
  • 快速重写中英文简历、项目经历和求职材料。
  • 把杂乱的报告草稿整理成可直接发给老板或客户的版本。

🛠️ AI 驱动开发七阶段 / Skills For Real Engineers
一句话

Matt Pocock 的开源工程技能材料，把 AI 驱动开发拆成 Grill、Research、Prototype、PRD、Issues、Implement、Review 七个阶段。

链接

https://github.com/mattpocock/skills ↗

怎么玩

打开仓库后从目录中找到与 AI 开发流程相关的章节，先不要急着复制模板，而是拿一个真实的小需求做演练。第一步 Grill：让 AI 追问需求边界，输出问题陈述；第二步 Research：让它查已有代码和相关方案；第三步 Prototype：用最小 Demo 验证；随后让它生成 PRD、拆 Issue、实现代码，最后进入 Review。你可以把七步做成 Claude Code/Codex 的固定提示词或项目 Skill，每次新需求都按阶段推进，而不是直接让模型「写代码」。

为什么值得关注

AI Coding 的质量问题往往不是模型不会写，而是缺少需求澄清、研究、验收和审查的过程。这个七阶段框架把传统工程纪律重新包装成适合 Agent 协作的流程，能显著降低一次性生成大量不可验证代码的风险。

应用场景
  • 给团队制定 AI 辅助开发 SOP，统一从想法到 PR 的流程。
  • 把个人 vibe coding 改造成可复盘、可审查的工程闭环。

🛠️ 乔木 App Review Insights
一句话

一个开源 App Store 用户评价分析工具，输入 App 名称后自动抓取评论并用 LLM 总结成产品洞察。

链接

https://appreview.qiaomu.ai ↗

https://github.com/joeseesun/qiaomu-app-review-insights ↗

怎么玩

最快方式是直接打开在线体验 appreview.qiaomu.ai，输入一个你关心的 App 名称，选择预设的国家、免费榜或付费榜数据，让系统抓取 App Store 评论并生成分析。每天限查 3 个 App，适合先验证效果；如果想研究实现或自部署，可以进入 GitHub 仓库，按 README 配置依赖、模型 API 和抓取参数。建议拿竞品 App 做测试，重点看差评聚类、用户高频需求、付费阻碍和功能建议，再把结果整理进产品 Roadmap。

为什么值得关注

App 评论是产品经理最容易忽略但极有价值的一手数据。这个工具把抓取、清洗、聚类和 LLM 解读串成完整流程，而且代码开源，适合学习「垂直场景数据 + LLM 分析」如何做成实用小产品。

应用场景
  • 分析竞品 App 的用户痛点、差评原因和高频功能诉求。
  • 给独立开发者做上架后评论监控和版本迭代依据。

一个 all-in-one 的 Claude、Codex 本地代理网关，支持 OAuth 登录、日志计费、隐私过滤和缓存预警。

🛠️ ZCode
一句话

智谱推出的类 Codex 编码客户端，支持 Windows、Mac，登录后可试用 GLM 5.2 进行 AI 编程。

链接

https://zcode.z.ai/cn ↗

怎么玩

访问 zcode.z.ai/cn，按系统选择 Windows 或 Mac 安装包；Mac 用户注意区分 Intel 与 Apple Silicon 版本，Linux 目前需要通过内测群获取。安装后使用 Google 或官方支持的方式登录，创建或打开一个真实代码项目，先让 ZCode 做低风险任务，比如解释项目结构、修一个小 bug、生成测试或重构单个函数。测试时建议开启 git，所有修改先看 diff 再接受，并对比你平时使用 Codex、Claude Code 的速度、上下文理解和补丁质量。

为什么值得关注

国内厂商正在把大模型能力从网页聊天推进到本地开发客户端。ZCode 的价值不只在模型本身，还在于它能否提供好用的项目索引、文件修改、命令执行和审查体验；如果 GLM 5.2 免费额度足够，对开发者是一个值得横向比较的新选择。

应用场景
  • 在本地项目中尝试国产模型驱动的代码修改、解释和测试生成。
  • 对比 Codex、Claude Code、ZCode 在同一任务上的补丁质量和交互体验。

🛠️ Agentic Detection / Perceptron Demo
一句话

一个视觉检测 Demo，用自然语言描述目标后，AI 能在图中精确框出对象，且不需要提前训练类别。

链接

https://perceptron.inc/demo ↗

怎么玩

打开在线 Demo 后上传一张包含多种物体或人物的照片，先用简单提示测试，例如「找出所有杯子」「圈出坐着的人」。随后尝试更复杂的物理关系或语义目标，例如「找出可能会滚落的物体」「找出桌面上不属于办公用品的东西」。它支持三种查询方式：不指定标签时全量盘点画面，给一组自然语言清单时逐类定位，用样例图时寻找更多相似对象并可加入反例。建议用自己的真实场景图测试边界能力，而不是只看官方样例。

为什么值得关注

传统目标检测依赖固定类别和训练数据，而 Agentic Detection 更像把视觉定位、语言理解和推理结合起来。它适合做「开放词表检测」和「任务驱动视觉理解」，未来可接入机器人、质检、安防或相册管理等场景。

应用场景
  • 给图片、视频帧做自然语言目标搜索和批量标注。
  • 在机器人或工业质检场景中按任务描述寻找异常物体。

🛠️ Text-to-Lottie Skill
一句话

Diffusion Studio 的开源 Lottie 生成工具/Skill，让 Agent 根据文字或真实数据生成 Lottie 动画。

链接

https://github.com/diffusionstudio/lottie ↗

怎么玩

进入 GitHub 仓库后查看 README，按说明安装依赖并运行示例。最简单的玩法是给 Agent 一段文字动画需求，例如「生成一个 AI Daily Digest 标题出现、数据点依次弹出的 5 秒 Lottie」，让它输出 Lottie JSON，再用网页播放器、LottieFiles 或项目自带预览方式查看效果。进一步可以把真实数据喂给它，比如增长曲线、排名变化、步骤流程，让 Agent 生成可嵌入网页或 App 的动态图表。生成后重点检查 JSON 大小、兼容性和动画节奏。

为什么值得关注

Lottie 是产品界常用的轻量动画格式，但手工制作门槛不低。Text-to-Lottie 把 Agent 变成动画脚本生成器，适合快速做解释型动画、数据动画和界面微交互原型，让非设计师也能产出可嵌入产品的动态素材。

应用场景
  • 为网页、App、PPT 快速生成轻量级动效和解释动画。
  • 把真实业务数据转成可视化 Lottie 动画，用于报告或仪表盘。

🛠️ Clawhunt
一句话

一个 AI Agent 悬赏任务市场，用户发布复杂任务并挂赏金，由不同 Agent 接单、执行和验收。

链接

https://clawhunt.store ↗

https://clawhunt.store/community ↗

怎么玩

打开 clawhunt.store 后先浏览已有任务类型，理解它把「复杂问题」包装成悬赏单的方式。作为使用者，可以注册账号，发布一个边界清楚的小任务，比如优化 SQL、写一个 Skill、搭建工作流或修复脚本，并设置验收标准和赏金；作为开发者，则可以按平台要求注册自己的 Agent，填写名称、能力描述、使用模型和 Webhook 地址，通过能力测试后接入平台抢单。建议从低金额、可自动验收的任务试起，观察任务分发、执行结果和付款流程是否可靠。

为什么值得关注

如果 Agent 真的能作为独立劳动力参与任务市场，那么关键不再只是模型能力，而是任务描述、能力匹配、验收、支付和声誉系统。Clawhunt 把这些环节产品化，值得作为「Agent 经济」的早期样本观察和试玩。

应用场景
  • 把零散工程任务、数据处理任务或自动化需求外包给 Agent 尝试完成。
  • 给自研 Agent 接入任务市场，测试真实用户任务中的可靠性和收益模型。

📡 Claude Agent SDK 与 claude -p 订阅额度政策调整
事件

Anthropic 撤回原先把 Agent SDK、claude -p 等第三方使用改为 API 计费的安排，改为给订阅用户提供专用额度，Pro 用户每月约 20 美元，Max 5x 约 100 美元。

解读

这次调整对重度 Claude Code、Agent SDK 和第三方 Agent App 用户影响很大。此前如果第三方调用完全转向 API 计费，会显著抬高个人开发者和工具作者的使用成本；现在改成订阅内单独额度，相当于承认「非交互式 Agent 使用」已经是 Claude 订阅的重要场景，但又通过专用预算控制算力风险。

影响评估

短期看，Codex、OpenClaw、Hermes Agent、Pi 等依赖 Claude 订阅侧能力的工具会获得喘息空间，用户迁移压力降低。长期看，模型厂商会继续在订阅、API、Agent SDK 之间寻找计费边界，开发者需要关注额度、上下文和可用模型是否随时变化。

📡 Anthropic 隐私政策加入身份/年龄验证相关条款
事件

Anthropic 更新隐私政策，提到在特定情况下可能要求 Claude 免费版、Pro 和 Max 用户提供年龄或身份验证，生效日期指向 2026 年 7 月 8 日。

解读

这不是一个可直接上手的工具，但对 Claude 生态用户非常关键。身份验证条款通常与合规、安全、出口管制、未成年人保护和高风险使用场景有关。现在信息还不完整，不能简单理解为所有用户立即强制 KYC，但对跨区使用、团队共享账号和自动化 Agent 调用来说，未来出现验证弹窗或访问限制的概率会上升。

影响评估

如果 Claude 成为更多开发者 Agent 工作流的底层模型，账号验证政策会直接影响连续可用性。个人用户应提前准备替代模型和多供应商路由；工具作者则需要在产品里处理验证失败、额度变化和模型不可用的降级逻辑。
