一句话

一个把操作视频自动转成带截图、带说明文字的步骤操作文档的开源项目，可输出 Markdown 和 PDF。

链接

https://github.com/lxfater/video-to-doc-stepfun ↗

怎么玩

这个项目的 README 已经把使用流程写得比较全。先准备一段教程视频或录屏，然后按仓库说明安装 `ffmpeg` 和 `whisper`，再配置 `.env` 中的 StepFun API Key。默认模式是“字幕驱动”：Whisper 先把语音转成带时间戳字幕，AI 再根据字幕识别操作步骤，成本更低、速度更快；如果你的视频很多关键信息在画面里，也可以切到更贵的“上传视频给 AI 看画面”模式。仓库还提供了 Web 界面：启动 FastAPI 后端和 React 前端后，直接在网页上传视频，就能实时看处理进度、在线编辑 Markdown，并导出 PDF。最适合的第一轮体验，是拿一段你自己的软件演示、后台配置流程或教程录屏跑一遍，看它能否自动生成一份像产品 SOP 一样的文档。

为什么值得关注

GitHub README 已直接说明它会统一输出视频、字幕、截图、steps.json、Markdown 和 PDF，说明这不是只有概念的 demo，而是把“视频→文档”的完整流水线串起来了。再结合推文里作者同步开源代码，以及 StepFun Startup Program 页面写明它鼓励基于 Step 3.7 Flash 构建 agent 与工作流，可以看出这是一个很适合继续往产品化方向打磨的项目。

应用场景
  • 把产品演示、后台操作、培训录屏快速整理成 SOP、客服文档或内部培训手册。
  • 做知识付费、课程运营或企业培训时，把视频内容同步沉淀成文字版步骤指南。
