🛠️ ego lite
一句话

ego lite 是一款给人和 AI Agent 共用的 Chromium 浏览器，让 Agent 在独立 Space 里并行操作网页，同时复用你的真实登录态、Cookie、扩展和标签环境。

链接

https://lite.ego.app/ ↗

https://github.com/citrolabs/ego-lite ↗

怎么玩

访问官网或 GitHub，按 Mac 芯片下载对应 DMG：Apple Silicon 版是 https://cdn.ego.app/setup/macos/arm64/egolite.dmg，Intel 版是 https://cdn.ego.app/setup/macos/x64/egolite.dmg。安装后首次启动可选择迁移 Chrome 数据，包括书签、扩展、Cookie、登录会话等；之后在支持技能的 Agent CLI 中使用 `ego-browser`，例如输入 `/ego-browser` 后让它打开网页、抓取信息、填写表单或执行多步页面任务。官方说明里还给了“让 agent follow @ego_agent on x.com”的示例，Agent 会在自己的 Space 打开页面、读取 Snapshot、执行操作，并且不抢你的鼠标和当前标签页。

为什么值得关注

它不是给 Agent 临时开的无状态浏览器，而是把“日常浏览器”和“Agent 自动化运行时”合在一起：Agent 可以用真实登录态处理 Gmail、Notion、LinkedIn、内部后台等需要认证的页面，同时通过语义 Snapshot、click/fill/navigate/capture 等工具减少逐步截图式操作的 token 和重试成本，对网页任务自动化非常关键。

应用场景
  • 让 Codex、Claude Code、Cursor 等外部 Agent 登录并操作后台、CRM、文档系统或数据面板。
  • 并行跑多个网页信息整理、表单录入、线索抓取任务，同时自己继续正常浏览。

腾讯 EdgeOne Makers Tools 是一套官方开源 AI Agent Skills，帮助 Agent 开发、部署和管理 EdgeOne Makers 上的 Web 应用、边缘函数、云函数、KV/Blob 存储和 AI Agent 项目。

链接

https://github.com/TencentEdgeOne/edgeone-makers-tools ↗

https://pages.edgeone.ai/document/pages-skills ↗

怎么玩

打开 GitHub 仓库可以看到 `makers-agents`、`makers-deploy`、`makers-edge-functions`、`makers-cloud-functions`、`makers-storage`、`makers-middleware`、`makers-cli`、`makers-recipes` 等技能目录。文档给出的命令行安装方式是 `npx skills add TencentEdgeOne/edgeone-pages-skills`，也可以让支持 Skills 的 AI 编程工具直接安装仓库。装好后对 Agent 说“Deploy this Next.js project and give me the preview URL”或“Build an AI chat agent on EdgeOne Makers”，它会按 Skill 流程检查环境、安装 CLI、登录、构建、上传并返回公开 URL。

为什么值得关注

这类平台把云部署知识写成 Agent 可读取的技能，而不是只提供传统文档。EdgeOne 的 Skill 覆盖部署、边缘函数、云函数、存储、中间件和 Agent 框架，意味着 Agent 不只是写代码，还能按平台规范选择运行时、放置目录、调试本地服务并完成上线，降低了从原型到可访问应用的摩擦。

应用场景
  • 把 Next.js、Nuxt、Vike 或静态项目通过一句话交给 Agent 部署到 EdgeOne Makers。

🛠️ AndroMeld
一句话

AndroMeld 是一款 macOS 工具，可把 Android 手机原生挂载到 Finder，并提供屏幕镜像、剪贴板同步、Mac 输入法输入和 USB/Wireless ADB 连接。

链接

https://andromeld.catchingnow.com ↗

https://meta.appinn.net/t/topic/87295 ↗

怎么玩

访问官网点击 `Direct Download` 下载 DMG，或从 Mac App Store 安装。安装后在 Android 手机上开启开发者选项里的 USB Debugging，用 USB 连接可获得更稳定和高带宽的体验；也可以使用 Wireless ADB 在局域网无线连接。连接成功后，Android 存储会出现在 Finder 侧边栏，支持拖拽文件、Mac 快捷键和空格 Quick Look；需要操作手机时打开镜像窗口，可以用 Mac 键盘和输入法直接在 Android App 中输入文字，并在两端复制粘贴文本或图片。

为什么值得关注

它试图给 Android + Mac 用户补上 Apple Continuity 式体验，而且不是简单投屏：页面明确提到通过 ADB 协议连接、本地点对点传输、Finder 原生文件映射、低延迟屏幕和音频投射、剪贴板双向同步。这对同时使用 Mac 和 Android 的开发者、内容创作者、测试人员非常实用。

应用场景
  • 在 Mac 上快速管理 Android 手机文件、相册、视频或 APK，不再反复用数据线传输工具。
  • 做 Android App 测试、演示或直播时，把手机画面投到 Mac 并用键盘输入。

🛠️ Apple container
一句话

Apple 开源的 `container` 是一个在 macOS 上创建和运行 Linux 容器的命令行工具，以轻量虚拟机方式运行 OCI 镜像，使用 Swift 编写并针对 Apple Silicon 优化。

链接

https://github.com/apple/container ↗

怎么玩

先确认机器满足要求：Apple Silicon Mac，并运行 macOS 26。进入 GitHub 仓库的 Releases 页面下载最新签名安装包，双击 pkg 按安装器提示完成安装，文件会放到 `/usr/local`。安装后可按 README/文档启动系统服务，再使用 `container` 拉取、运行或构建 OCI 兼容镜像；仓库也提供 `update-container.sh` 和 `uninstall-container.sh` 这类辅助脚本用于升级、降级和卸载。如果想从源码构建，可继续阅读仓库里的 `BUILDING.md`。

为什么值得关注

Mac 上跑 Linux 容器长期依赖 Docker Desktop 或第三方虚拟化层，Apple 自己给出 Swift 实现、Apple Silicon 优化、OCI 镜像兼容的工具，代表 macOS 26 的虚拟化与网络能力正在直接面向开发者开放。它仍处于活跃开发、1.0 前可能破坏性变更，但已经很值得容器和本地开发环境用户试验。

应用场景
  • 在 Apple Silicon Mac 上用更原生的方式运行 Linux 容器镜像和开发服务。
  • 研究 Apple Containerization Swift 包、macOS 虚拟化与 OCI 容器运行时实现。

🛠️ Google Workspace CLI（gws）
一句话

Google Workspace CLI 是一个开源命令行工具，用 `gws` 统一操作 Drive、Gmail、Calendar、Sheets、Docs、Chat、Admin 等 Google Workspace API，并输出结构化 JSON 方便 Agent 使用。

链接

https://github.com/googleworkspace/cli ↗

怎么玩

进入 GitHub 仓库下载适合系统的最新 Release，把 `gws` 二进制放到 `$PATH`；也可以通过 npm 包 `@googleworkspace/cli`、Homebrew 或 Nix flake 安装。若本机已有并已登录 `gcloud`，README 推荐用 `gws auth setup` 快速配置认证；如果没有 `gcloud`，可走手动 OAuth、环境变量 `GOOGLE_WORKSPACE_CLI_TOKEN` 或 `GOOGLE_WORKSPACE_CLI_CREDENTIALS_FILE`。配置完成后用各资源的 `--help` 查看命令，先用 `--dry-run` 预览请求，再执行 Drive、Gmail、Calendar 等 API 操作。

为什么值得关注

它的核心亮点是命令面不是静态写死，而是运行时读取 Google Discovery Service 动态生成；当 Workspace 新增 API endpoint 时，工具理论上可以自动获得新能力。再加上 JSON 输出、自动分页、dry-run 和大量 Agent Skills，它很适合作为 AI Agent 操作企业 Workspace 的底层工具。

应用场景
  • 用脚本或 Agent 批量整理 Google Drive、查询 Gmail、创建 Calendar 日程或操作 Sheets。
  • 为内部自动化搭建统一 Workspace CLI，而不是为每个 Google API 手写 curl 和认证逻辑。

🛠️ Transitions.dev
一句话

Transitions.dev 是一个网页过渡动画组件库，提供可复制粘贴的卡片缩放、数字翻转、通知徽章、菜单、Modal、页面切换、错误抖动等动效示例，还支持安装 Skill 让 Agent 帮网页加动画。

链接

https://transitions.dev/ ↗

怎么玩

打开 transitions.dev，直接浏览页面上的动画样例，比如 Card resize、Number pop-in、Notification badge、Menu dropdown、Modal open/close、Page side-by-side、Error state shake 等。看到合适的过渡效果后复制对应代码到你的前端项目中调试。推文还给出 Agent Skill 安装命令：`npx skills add jakubantalik/transitions.dev`，装好后可以让支持 Skills 的编码 Agent 阅读该技能，并根据你的网页组件结构自动挑选和添加更合适的动效。

为什么值得关注

多数动效库要么太大、要么偏设计展示，Transitions.dev 把常见 Web App 微交互拆成可直接复制的“小过渡”，对 AI 辅助前端开发尤其友好。配合 Skill 后，Agent 可以在改 UI 时不只生成静态布局，还能遵循现成动效模式补上可感知的质量细节。

应用场景
  • 给 SaaS 面板、表单、菜单、Modal、通知和数据卡片快速加上更自然的交互动效。
  • 作为前端 Agent 的动效参考库，让它按现成模式改进网页体验。

🛠️ WebDAV Lite
一句话

WebDAV Lite 是一款 macOS 菜单栏小工具，可以一键把任意本地文件夹变成局域网 WebDAV 服务器，方便 iPad、Apple TV、Infuse、VLC、Finder 等访问。

链接

https://www.appinn.com/webdav-lite-for-macos/ ↗

https://apps.apple.com/app/webdav-lite-lan-file-server/id6779282964 ↗

怎么玩

从 App Store 安装 WebDAV Lite（小众软件文章提示非国区），启动后它会常驻菜单栏。选择想共享的 Mac 文件夹，一键开启局域网 WebDAV 服务；在 iPad、Apple TV、Infuse、VLC 或 Finder 中添加该 WebDAV 地址即可访问、同步或串流文件。文章还提示目前是应用内购买限免：进入 App 内购买页面，看到 `$0.00` 可永久解锁 Full Unlock。需要安全访问时开启 HTTPS 加密和密码保护。

应用场景
  • 把 Mac 上的高清视频目录共享给 Apple TV、iPad、Infuse 或 VLC 播放。

📡 Codex Remote 官方工程实践指南
事件

OpenAI Developers Blog 发布《Mastering Codex Remote for engineering》，强调手机不是小终端，而是远程控制工程任务的 control plane。

解读

这篇指南把 Codex Remote 的使用方式讲得很清楚：代码仍然运行在 Mac、Windows、devbox 或其他已连接主机上，手机负责选择仓库、工作区、分支、是否新建 worktree、是否排队或中途 steer、是否批准命令和审查改动。它不是让工程师在手机上深度读代码，而是把“启动、转向、批准、归档、复盘”这些阻塞 Agent 工作流的决策移动化。

影响评估

如果这种模式成熟，AI 编程工具的竞争点会从“模型会不会写代码”扩展到“能不能把长任务稳定挂在真实开发机上，并让人随时低成本介入”。远程控制、队列/转向语义、权限批准、worktree 隔离和手机端 review 都会变成 Agent IDE 的标准能力。

📡 Google Workspace CLI 作者离职争议
事件

多位博主讨论 Google Workspace CLI 开源发布后，原作者 Justin Poehnelt 被 Google 解雇一事。

解读

📡 Claude Tag 异步协作形态讨论
事件
