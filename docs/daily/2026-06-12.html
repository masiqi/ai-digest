🛠️ Pake — 网页转桌面应用
一句话

把任何网页打包成轻量级原生桌面应用，一条命令搞定，50K GitHub 星标。

链接

https://github.com/tw93/pake ↗

怎么玩

安装 Node.js 后执行 `npm install -g pake` 或直接 `npx pake <url>`，支持 macOS/Windows/Linux。最新版加入了 --enable-find 页面内查找、右键图片/视频下载（即使 CSP 严格页面也能用）、应用名支持点号。例如 `pake https://chat.openai.com --name ChatGPT` 就能得到一个原生 ChatGPT 桌面客户端，还可以自定义图标、窗口大小、注入 CSS/JS。也支持通过 Rust 后端编译成更小的二进制包。

为什么值得关注

用 Tauri 而不是 Electron，包体积小得多（通常 < 5MB），内存占用极低。50K 星标说明社区认可度极高，50+ 个现成的流行网站打包方案可以直接用。最新更新解决了 CSP 页面下载和页面内搜索两个长期痛点。

应用场景
  • 把内部工具网页打包成桌面应用分发给非技术同事
  • 为常用 SaaS 工具创建独立窗口，避免浏览器标签页混乱

🛠️ DiffusionGemma — Google 文本扩散推理加速
一句话

Google 开源 26B MoE 模型，用文本扩散替代自回归生成，H100 上跑出 1000+ tokens/秒。

链接

https://blog.google/technology/developers-tools/diffusion-gemma-faster-text-generation/ ↗

怎么玩

从 HuggingFace 下载模型权重（Google 官方发布），用 transformers 库加载。核心创新是同时生成 256 个 token 而非逐个生成，把瓶颈从内存带宽转移到计算。26B 总参数但 MoE 架构仅激活 3.8B，单卡 H100 即可运行推理。适合代码补全、批量文本生成等对延迟敏感的场景。官方提供了 Colab notebook 可直接体验。

为什么值得关注

文本扩散是自回归之外最有潜力的生成范式，4 倍推理加速意味着实时应用的成本大幅降低。Google 第一次把扩散文本模型做到可用的质量和规模，定位虽然还是实验级，但方向意义重大。

应用场景
  • 本地低延迟代码补全服务，替代云端 API
  • 批量文本生成场景（如数据增强、评测集构造）

🛠️ Datasette 1.0a33 + API Explorer
一句话

Simon Willison 发布 Datasette 新版本，引入 `?_extra=` JSON API 并配套可视化浏览器工具。

链接

https://datasette.io ↗

https://tools.simonwillison.net/datasette-extras-explorer ↗

怎么玩

`pip install datasette` 或 `uv pip install datasette`，然后在任意 SQLite 数据库目录下运行 `datasette mydb.db`。新版核心是 `?_extra=` 查询参数，可以在一次请求中获取表格的列元数据、外键关系、行数统计等附加信息，减少前端多次请求。配套的 API Explorer 工具（由 Fable 5 + GPT-5.5 协助开发）让你在浏览器里交互式探索所有 `?_extra=` 参数的效果。大部分新代码由 Claude Fable 5 辅助编写。

为什么值得关注

Datasette 是「快速把 SQLite 数据库变成可探索的网站和 API」的标杆工具。新版 `?_extra=` 机制大幅简化了前端开发，是 Simon 探索「AI 辅助开发 + 开放数据」结合的又一个里程碑。

应用场景
  • 把 CSV/JSON 数据导入 SQLite 后用 Datasette 即时发布成可查询的 Web API
  • 与 LLM 结合做数据分析——Datasette 提供结构化查询接口，LLM 写 SQL
