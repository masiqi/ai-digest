🛠️ ChatGPT for PowerPoint（OpenAI 官方插件）
一句话

直接在 PowerPoint 右侧对话框里用自然语言生成/改进/总结 PPT，输出完全可编辑

怎么玩

在 PowerPoint 中点击「插入」→「获取加载项」→ 搜索 ChatGPT 安装官方插件。安装后右侧出现 ChatGPT 对话框，你可以：(1) 用大白话描述需求直接生成新 PPT；(2) 让它读现有 PPT 逐页改进文案和排版；(3) 把会议记录或数据表粘贴进来，自动整理成展示内容；(4) 让它总结整份 PPT 的故事线。所有输出都是 PowerPoint 原生可编辑格式，不是死图片。需要 ChatGPT Plus/Pro 订阅。

为什么值得关注

这是 OpenAI 首次将 ChatGPT 原生嵌入 Office 工作流，不再是浏览器里复制粘贴——生成的每张幻灯片、每段文字都能直接在 PPT 里调整，打通了 AI 内容生成到生产力工具的最后一步

应用场景
  • 周报/月报：把零散数据和要点扔进去，一键生成汇报 PPT
  • 会议纪要转汇报：粘贴会议记录，自动提炼要点并排版成展示稿

🛠️ Codex Appshots：双击 Command 把当前窗口上下文发给 AI
一句话

在 Mac 任意应用中双击 Command 键，Codex 截取当前窗口截图+读取所有文字内容（含屏幕外部分），直接开始对话

链接

https://codex.openai.com（需安装 ↗

怎么玩

安装最新版 Codex Mac 客户端后，在任意应用（浏览器、VS Code、Figma、文档编辑器等）中工作，遇到需要 AI 帮忙的地方，双击左右 Command 键（⌘⌘，可在设置中修改快捷键）。Codex 会自动截取当前窗口，不仅抓截图，还会提取窗口内所有文字（包括没显示在屏幕上的滚动部分），然后你可以直接提问：「这段代码有什么问题？」「帮我改写这段文案」「这个设计的配色怎么样？」。更进阶的是 /goal 模式，可以给 Codex 设定目标让它跨应用完成任务。

为什么值得关注

这比传统截图+手动粘贴高效太多——窗口上下文一键送达，Codex 还能读到屏幕外的文字，相当于 AI 有了「读屏能力」。配合 /goal 模式，AI 从聊天助手进化成真正替你操作电脑的代理

应用场景
  • 开发调试：在 IDE 里双击 Command，让 Codex 分析当前文件的问题
  • 文档协作：在 Notion/飞书里双击，让 AI 帮你润色或翻译整篇文档

🛠️ Codex Locked Use：Mac 锁屏后 AI 继续干活，手机远程指挥
一句话

Codex 通过苹果官方 Authorization Plug-in 机制在锁屏状态下临时解锁 Mac 干活，屏幕保持黑屏，干完自动锁回

怎么玩

在 Codex Mac 客户端设置中启用「Computer Use」和「Locked Use」功能。之后你可以：锁屏离开工位→用手机上的 Codex App 下发任务（比如「跑完这个测试套件，把结果发我邮箱」）→Codex 在锁屏状态下通过苹果官方 API 临时解锁 Mac 执行任务→屏幕全程保持黑屏，隐私安全→任务完成后自动重新锁屏。手机端还能收到编译完成、任务结束的实时推送通知。这用的是苹果 macOS 原生的 Authorization Plug-in 机制，不是第三方 hack。

为什么值得关注

这解决了「AI 需要跑长任务但人不能一直守着电脑」的核心痛点。手机远程下达任务 + 锁屏安全执行 + 自动锁回，真正实现了「AI 替你加班」

应用场景
  • 睡前让 Codex 跑测试/训练/数据爬取，早上看结果
  • 通勤路上用手机指挥家里的 Mac 继续开发工作

🛠️ Chrome DevTools for Agents 1.0：让 AI Agent 真正「看见」浏览器
一句话

Google 官方发布的 Agent 浏览器调试工具，支持 MCP Server / CLI / Agent SDK 三种接入方式，让 AI 能像真实用户一样验证和调试 Web 代码

链接

https://developer.chrome.com/blog/devtools-for-agents-v1 ↗

怎么玩

三种接入方式——(1) MCP Server：在你的 AI 工具（Claude Code/Codex/Hermes 等支持 MCP 的客户端）中配置 Chrome DevTools MCP Server，Agent 即可操作真实浏览器、查看页面行为、检查 console 输出；(2) CLI：更省 Token 的方式，Agent 把操作打包成脚本批量在浏览器中执行，适合高频自动化测试；(3) Agent SDK：直接在你的 Agent 代码中集成 DevTools 能力。安装后 Agent 能做的事情包括：打开网页检查渲染效果、读取 console 错误、截取页面截图、执行 JS 并获取返回值。

为什么值得关注

这是第一个官方级别的「Agent 浏览器眼睛」。之前 Agent 写前端代码只能盲写，看不到实际渲染效果和报错，现在有了完整的浏览器调试能力，Agent 开发 Web 应用的准确率将大幅提升

应用场景
  • Agent 写前端代码后自动打开浏览器验证 UI 效果和 console 报错
  • 自动化端到端测试：Agent 模拟用户操作路径并检查页面行为

应用场景

🛠️ MLX-Vulkan：让 MLX 跑在所有平台上
一句话

把 Apple 的 MLX 框架移植到 Vulkan 后端，意味着 MLX 模型能在 Windows/Linux/Android 等任意支持 Vulkan 的设备上运行

链接

https://github.com/goniz/mlx-vulkan ↗

怎么玩

git clone 后按 README 编译（需要 Vulkan SDK），编译完成后可以像使用原生 MLX 一样加载和推理模型，但计算后端从 Metal 换成了 Vulkan。这意味着原本只能在 Apple Silicon 上跑的 MLX 模型（如 llama.cpp 的 MLX 版本、Stable Diffusion MLX 等），现在可以在 NVIDIA/AMD 显卡甚至 Android 手机上运行。目前处于早期阶段，部分算子仍在移植中。

为什么值得关注

MLX 是 Apple 生态最优秀的本地 AI 推理框架，MLX-Vulkan 打破了平台锁定。如果 Vulkan 后端成熟，MLX 模型将真正实现「一次编写，到处运行」，对端侧 AI 部署意义重大

应用场景
  • 在 Windows/Linux 工作站上运行原本 Mac 独占的 MLX 模型
  • 将 MLX 优化的模型部署到 Android 设备端
