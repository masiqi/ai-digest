🛠️ 微信公众号转 Markdown
一句话

输入微信公众号文章 URL，就能把文章转换成 Markdown，并自动下载文章里的图片。

链接

https://github.com/jackwener/wechat-article-to-markdown ↗

怎么玩

先打开 GitHub 仓库查看 README，按项目要求准备 Python/Node 环境和浏览器依赖；如果你只是想试验，找一篇公开的公众号文章 URL，按 README 的命令把 URL 作为参数传入，运行后检查输出目录里的 Markdown 文件和图片资源。它还提到会配合 camoufox 这类反检测浏览器，所以遇到微信页面无法正常抓取时，可以先安装 camoufox，再重新执行转换流程。转换完成后，把 Markdown 放进 Obsidian、Typora、Notion 导入器或自己的 RAG 知识库里测试效果。

为什么值得关注

公众号内容长期被困在网页和移动端里，复制排版、图片保存、归档都很麻烦。这个项目把反爬访问、正文抽取、图片下载和 Markdown 输出串成一条链路，适合做个人知识库、内容再编辑、文章转 PPT 或企业内部资料沉淀的前置工具。

应用场景
  • 把高质量公众号文章批量归档到本地知识库。
  • 将公众号长文转换为 PPT、PDF、RAG 文档或二次编辑素材。

🛠️ camoufox
一句话

一个面向自动化抓取场景的反检测浏览器，可用于绕过常规浏览器指纹识别。

链接

https://github.com/daijro/camoufox ↗

怎么玩

进入 GitHub 仓库按 README 安装，通常会先准备 Python 环境，再安装 camoufox 包和浏览器二进制。安装后可以把它当成自动化浏览器后端，用在 Playwright/脚本抓取流程里；最简单的玩法是先跑仓库示例，确认能打开目标网页，再把原先脚本里的普通 Chromium 启动逻辑替换为 camoufox。和微信公众号转 Markdown 项目一起用时，可以先用 camoufox 打开目标文章，看正文和图片是否完整渲染，再执行内容抽取。

为什么值得关注

越来越多站点对 Playwright、Selenium、Headless Chrome 做指纹检测，普通自动化脚本经常还没抓到内容就被拦截。camoufox 的价值在于降低自动化浏览器被识别的概率，让个人归档、内容迁移、内部数据采集这类合法自动化流程更稳定。

应用场景
  • 给网页归档、资料抽取、内容迁移脚本提供更稳的浏览器环境。
  • 调试被常规 Headless Chrome 阻断的自动化任务。

🛠️ Deep Agents 实战
一句话

Datawhale 开源的 Deep Agents 教程，围绕 LangChain/LangGraph 生态讲解如何构建真实 Agent 应用。

链接

https://github.com/datawhalechina/deepagents-in-action ↗

怎么玩

打开 GitHub 仓库后先从目录和 README 看课程结构，按章节安装 Python 依赖并配置所需模型 API Key。建议不要一口气读完，先 clone 到本地，跟着第一章跑通最小 Agent 示例，再逐章实验工具调用、状态管理、长任务拆解和多步骤执行。每完成一节，就把示例改成自己的任务，比如网页总结、资料检索、客服问答或代码助手，这样能真正理解 LangGraph 编排和 Deep Agents Harness 的边界。

为什么值得关注

Agent 现在最大的问题不是“会不会调用模型”，而是如何把上下文、工具、记忆、状态和评估组织成稳定工程。这个教程来自 LangChain 官方认证大使，重点放在可落地工程实践，比单纯 prompt 合集更适合开发者系统学习。

应用场景
  • 学习 LangChain/LangGraph 并构建自己的生产级 Agent 原型。
  • 给团队做 Agent 工程培训或内部技术分享。

🛠️ Cloudflare Workers 临时账号 / temporary deploy
一句话

Cloudflare 给 AI Agent 提供临时部署账号，Agent 可先部署验证 Worker，再由真人认领资源。

链接

https://blog.cloudflare.com/temporary-accounts/ ↗

怎么玩

先把 Wrangler 更新到 4.102.0 或更新版本，然后在一个 Cloudflare Workers 项目目录里运行 `wrangler deploy --temporary`。命令会创建短期账户和自动生成的 API Token，把 Worker 部署到 workers.dev 子域名，开发者或 Agent 可以在 60 分钟有效期内验证代码、KV、D1 等资源是否正常。确认没问题后，打开 Agent 返回的 claim/认领链接，把这些临时资源绑定到你自己的 Cloudflare 账号，最后再按常规方式纳入项目管理。

为什么值得关注

过去 Agent 写完代码后，部署环节经常卡在人类账号、API Token、权限授权上。临时账号把“先部署验证”和“后绑定归属”拆开，大幅降低了自动化开发闭环的摩擦，是 Agent 从写代码走向自主交付的重要基础设施。

应用场景
  • 让 Claude Code、Codex 或自研 Agent 自动部署 Cloudflare Worker Demo。
  • 在不给长期 API Token 的情况下验证第三方生成的边缘应用。

📡 Codex Handoff 支持跨设备任务迁移
事件

Codex 上线 Handoff，可把本地进行中的编码任务迁移到远程服务器继续执行，再切回本地。

解读

这个功能解决的是 Agent 编程里很真实的痛点：长任务不能被笔记本合盖、网络变化或本地算力限制打断。Handoff 的关键不是简单同步文件，而是迁移“任务上下文 + 代码状态 + 执行环境”。它让 Codex 更像一个可持续工作的远程开发代理，而不是只能依附在当前终端里的聊天助手。

影响评估

如果这种跨设备迁移稳定可用，AI 编程工作流会更接近云端 IDE + 本地入口的混合模式。开发者可以在通勤、会议和夜间把任务交给远程 Agent 跑，回来再接管审查，团队也可能围绕远程 Agent 池建立新的开发基础设施。
