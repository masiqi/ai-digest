🛠️ Google Earth 飞行模拟器（网页版）
一句话

Google 地球内建的飞行模拟器现已面向全球所有用户免费开放，在浏览器里就能操控飞机飞越全球任意地标。

链接

https://goo.gle/4fBYnWO ↗

怎么玩

打开 Google Earth 网页版（earth.google.com），点击左下角菜单图标找到「飞行模拟器」或直接通过上述短链接进入。选择一架飞机（F-16 战斗机或 SR22 螺旋桨飞机），然后在全球任意地点点击「开始飞行」即可。用鼠标控制俯仰和方向，键盘方向键也能辅助操控。建议先从自己家上空起飞，再尝试飞跃大峡谷、阿尔卑斯山或迪拜棕榈岛等经典地标。操作手感需要适应几分钟，小互反馈鼠标操控有点晕，多练习就好。

为什么值得关注

这不是第三方简陋模拟器，而是 Google Earth 官方整合的 3D 飞行体验，底层依托 Google 地球的全球高精度 3D 地形和建筑数据。在浏览器里流畅运行无需下载任何客户端，技术实现依赖 WebGL 和 Cesium/MapGL 等现代前端图形栈，是地理信息可视化与游戏化体验的绝佳交叉案例。

应用场景
  • 地理教学——让学生飞越课本上的每个地名
  • 旅行规划——提前从空中预览目的地地形和城市布局

🛠️ FanBox——Agent 容器框架
一句话

花叔在 Fable 5 可用的 72 小时内疯狂迭代出的 Agent 运行容器，支持多 Agent 沙箱隔离、工具链编排和持久状态管理，已开源。

链接

https://github.com/alchaincyf/fanbox ↗

怎么玩

克隆仓库后 `pip install -r requirements.txt` 安装依赖。FanBox 核心概念是「沙箱容器」——每个 Agent 跑在独立命名空间内，互不干扰。配置 YAML 定义工具链（文件系统、网络、Shell、浏览器等），支持接入 Claude Code、Codex、Pi 等后端。启动后 Agent 自动读取任务描述并循环执行，中间状态保存在 .fanbox/ 目录下，支持断点续跑。花叔 3 天烧了 5 亿 token 迭代 20+ 版本，代码质量和架构设计经过实战检验。

为什么值得关注

Agent 编排框架目前各家闭源（Claude Code、Codex CLI），FanBox 是少见的开源通用 Agent 容器方案。沙箱隔离机制让多个 Agent 并行工作而不互相污染上下文，特别适合需要多 Agent 协作的复杂工程任务。Fable 5 谢幕之作，架构被花叔称为「太特么好用了」。

应用场景
  • 并行跑多个 Code Agent 处理大型 monorepo 的不同模块
  • 搭建自动化 CI Agent——PR 提交后自动 review + 修 bug + 跑测试

🛠️ Kami——AI 简历生成器
一句话

Tw93 开发的终端 AI 简历工具，输入 Markdown 素材一句话生成排版精美的专业简历，支持多种模板风格。

链接

https://github.com/tw93/Kami ↗

怎么玩

`npm install -g kami` 全局安装（或 `npx kami` 直接运行）。准备好自己的简历素材写成 Markdown 文件（教育经历、工作经验、项目经历、技能栈）。在终端执行 `kami init` 生成配置文件，然后运行 `/kami` 指令让 AI 读取你的 Markdown 并自动排版输出。支持交互式调整——觉得某段太长或风格不满意，直接说「把工作经历改成更简洁的风格」即可微调。Tw93 为简历场景单独优化了渲染效果，清晰整洁。

为什么值得关注

Kami 本质是 CLI + AI 的排版引擎，把「写简历」这个高频刚需变成了「准备素材→一句话指令→调两下→导出 PDF」。相比在 ChatGPT 里反复 prompt 调整格式，Kami 的模板化渲染更可控且输出一致。开源 MIT 协议，可以 fork 后定制自己的简历模板。

应用场景
  • 海投不同岗位时快速产出多版本定向简历
  • 帮朋友改简历——让他丢 Markdown 过来，30 秒出成品

🛠️ ZONOS2——开源 MoE TTS 模型
一句话

8B 总参数（0.9B 激活）的混合专家文本转语音模型，支持多语言和语音克隆，中文效果不错。

链接

https://huggingface.co/zyphra/zonos2 ↗

怎么玩

安装 `pip install zonos` 后，通过 Python API 调用。核心用法：① 基础 TTS——`model.speak("你好世界")` 直接生成中文语音；② 语音克隆——提供 5-10 秒参考音频，`model.clone("要说的文本", reference_audio="voice.wav")`，模型会模仿参考音色朗读新内容；③ 多语言切换——同一模型支持中/英/日/韩/法等十余种语言。0.9B 激活参数意味着在 M1/M2 Mac 上也能本地跑（约需 2-3GB 显存/内存）。

为什么值得关注

MoE 架构在 TTS 领域尚属少见，8B 总参数量大但推理只激活 0.9B，兼顾质量与速度。中文效果实测不错，语音克隆质量接近商业 API（如 ElevenLabs），且完全本地运行无需联网。开源 Apache 2.0 协议，可商用。

应用场景
  • 给自己播客或视频配 AI 旁白，不用买商业 TTS API
  • 用 5 秒录音克隆自己声音，做个性化语音助手

📡 Claude Fable 5 被美国政府出口管制下线
事件

美国政府以国家安全与出口管制为由，要求 Anthropic 停止向所有非美国公民提供 Fable 5 和 Mythos 5 模型访问。Anthropic 因无法区分用户国籍，一刀切下线全球所有非美国用户的权限，仅保留已验证美国身份的用户。

解读

Fable 5 上线仅 3 天即被封杀，被社区戏称为「AI 史上最短命的神模型」。讽刺的是，Anthropic CEO Dario Amodei 长期在华盛顿游说 AI 安全监管，最终监管回旋镖打到了自己身上。全球开发者社区反应激烈——Simon Willison 每分钟写脚本轮询 API 看自己何时被踢；花叔在 Fable 5 上烧了 5 亿 token 迭代 20 版 FanBox 后被迫中止；多个依赖 Fable 5 大规模重构的项目半途而废。

影响评估

中国开源模型获得结构性利好——智谱 GLM 5.2 发布、Kimi K2.7-Code 开源恰逢其时，大量海外开发者开始认真评估中国模型替代方案。这次事件将加速 AI 供应链的去美国化，欧洲和亚洲市场对非美模型的需求会爆发式增长。Anthropic 原定 6/22 的 Fable 5 Build Day 已降级为 Opus 4.8 Build Day。

📡 Kimi 开源 K2.7-Code 编码模型
事件

Kimi 发布 K2.7-Code，在 K2.6 基础上针对编程 Agent 专项优化，三项编码基准均有提升：Kimi Code Bench v2 +21.8%，长链路任务成功率更高，推理 token 更少。

📡 Vercel 发布 HarnessAgent
事件

Vercel 在 AI SDK 7 Canary 版中发布 HarnessAgent，提供统一 API 支持在 Claude Code、OpenAI Codex、Pi 等 agent 运行环境间无缝切换，同时提供沙箱（Sandbox）隔离机制。

解读

HarnessAgent 解决的核心痛点是 Agent 框架碎片化——开发者要在 Claude Code / Codex / Pi 之间反复切换，HarnessAgent 用一个统一接口抹平差异。Y11 贴出的用法显示安装非常简洁（npm install @ai-sdk/harness/agent），适合需要跨平台测试 Agent 性能的开发者。

影响评估

Vercel 入局 Agent 编排层，意味着 AI 开发工具链正在向标准化方向发展。如果 HarnessAgent 成为主流，可能会削弱单个 Agent 框架的锁定效应——开发者可以轻松把任务从 Claude Code 切到 Pi 或 Codex。
