🛠️ AppGroup

链接

怎么玩

🛠️ Cloudflare Workflows Saga Rollbacks
一句话

Cloudflare Workflows 新增 Saga Rollbacks，可在 `step.do()` 里直接声明失败后的补偿/回滚逻辑。

链接

https://developers.cloudflare.com/workflows/build/rules-of-workflows/#configuring-retries-and-rollbacks ↗

怎么玩

如果你在 Cloudflare Workers 里使用 Workflows，升级到支持 rollback 的写法后，把外部副作用步骤都包进 `step.do()`，并在最后一个 options 参数里加入 `rollback` 函数。例如扣款步骤可写成 `step.do("debit-bank-a", () => bankA.debit(...), { rollback: async ({ output }) => bankA.credit(..., output.id) })`，后续 credit 或 notify 步骤失败时，Workflow engine 会按完成步骤的相反顺序执行补偿。实际使用时要给正向操作和 rollback 操作都设计 idempotency key，避免重试导致重复扣款、重复退款或库存释放多次。

为什么值得关注

多步骤应用最难处理的不是“某一步失败”，而是前面步骤已经对支付、库存、邮件、第三方 API 产生了不可自动撤销的副作用。Saga Rollbacks 把补偿逻辑放在正向步骤旁边，减少巨大的 catch/unwind 代码块，也更适合构建可靠的异步业务流程。

应用场景
  • 支付、退款、库存锁定、订单确认等有外部副作用的 Durable workflow。
  • 用 Cloudflare Workflows 编排多 API 调用，并要求失败时自动补偿已完成步骤。
